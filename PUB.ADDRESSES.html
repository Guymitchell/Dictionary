Practice Manager Archive Enquiry/Details Screen

The ADDRESSES table corresponds to information added in the Add/Amend Address option of the CLIENT or INDIVIDUAL or ORGANISATION maintenance screens.

The fields available for completion are the same whether called from a CLIENT record or an INDIVIDUAL or ORGANISATION record. However, certain address types, viz Billing and Statement Address can be added only to a CLIENT record.



Field Notes

ADDR-TYPE corresponds to the radio button choice address type: Alt Correspondence = 1, Billing = 2, Statement = 3 or Other = 4, DX = 5. Only clients can have addresses of TYPE 2 or 3. For DX addresses the DX number is stored in ADD-ADDRESS-1 and DX exchange name is stored in ADD-ADDRESS-2.

TYPE corresponds to Client (CL), individual Contact (CO) or Organisation (OR). No other types of entity can have additional addresses.

SEQNO is a unique sequential number generated by the software to uniquely identify an address and has linking or reporting purpose.

Address Fields in CLIDB, CONTACTS and ORGANISATION tables

Not all addresses are kept in the ADDRESSES table.

For a contact or organisation the first or only address is stored in either the CONTACTS or ORGANISATION tables itself. Only second and subsequent addresses are kept in this ADDRESSES table.

For a client created from one or more individual contacts or organisations the address of the first or sole contact or organisation will be copied automatically into those address fields of the client’s CLIDB record that are prefixed with an “A”. Likewise the address of a second contact or organisation will be copied into those address fields of the client’s CLIDB record that are prefixed with a “B”.

Adding an Address to an existing Client created without an Address

For all Clients without a “primary” address ie an address that shows in the Address column of the General Search screen care should be taken when adding an address for the first time: if you add an address via the Add Address button on the Client Enquiry screen then the first such address will be kept in the CONTACTS or ORGANISATION table only, and second and subsequent addresses will be in ADDRESSES only. To keep an address in CLIDB you must add the primary address via the Organisation Add/Amend or Contact Add/Amend screen as illustrated above.

Adding further Client Addresses beyond the first two

New addresses added from the Client Enquiry screen will be kept in the ADDRESSES table

Access to CLIDB.B-ADDRESS and CLIDB.B-POSTCODE fields

The “B” address and postcode fields are used for the primary address of a contact/organisation that is added as the second party of the client. Updating the contact or organisation address will automatically update the CLIDB.B-ADDRESS and CLIDB.B-POSTCODE fields where the contact or organisation is the second party of the client.

Suggested Linking

All addresses for clients

A.CLIDB.CL-CODE -> PUB.ADDRESSES.REFERENCE WHERE PUB.ADDRESSES.TYPE = "CL" Use left outer join

We have suggested linking these tables with a left outer join to ensure that clients who do not have an additional address defined are still include in the query results.
