Practice Manager Interface

This table holds nominal code maintenance information as defined in Practice Manager nominal codes maintenance screen.






Field Notes

ANALYSE is No by default for new nominal codes. Some inbuilt control codes such as X08 and X20 are preset to Yes by SOS on installation of Practice Manager.

BALANCING-CODE is an optional default cashbook for postings to this nominal analysis code and should be ignored. Never assume the value in this field has been used for an actual transaction – look instead at the NOMTRANS.CONTRA-CODE field for the code of the cashbook that was actually used.

BAL-PL cannot be # alone as this symbol demands a suffix to indicate the type of bank, i.e. "C", "D" or "O" to signal "Client", "Deposit" or "Office".

CLEAR-TO is an alternative year-end clear down nominal code to the implied default of X40, so would be blank in most or all cases.

DR-CR simply defines whether a nominal is usually a debit or credit item.



Linking and Application

NOMCODES usually partners NOMACCS and other tables such as NOMTRANS so as to provide a description of an analysis code or cashbook that featured in a transaction.


This example could provide a list of postings grouped and totalled by analysis code and/or cashbook:

NOMTRANS.NL-CODE -> NOMCODES.CODE


This example ignores details and provides a summary of balances by analysis code and/or cashbook:

NOMACCS.NL-CODE -> NOMCODES.CODE
