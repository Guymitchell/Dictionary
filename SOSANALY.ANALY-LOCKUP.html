Table Name:SOSANALY.ANALY-LOCKUP

Description: If installed, this view will retrieve data from the SOS data warehouse, returning bills delivered, debtors, time recorded, WIP value, debtor lock up days and WIP lock up days at various summary levels


Field List:

Field NameData TypeDescription
BILL-LOCKUP-DAYSdecimalBill (debtor) lock up days for this summary level / time span
BILL-LOCKUP-ESTlogical (true/false)Set to YES if the debtor lock up days calculation on this record is an estimate
BILL-VALUEdecimalTotal value (gross) of bills delivered for this summary level / time span
EXCLUDE-FROM-CALCULATIONlogical (true/false)Usually set to NO, indicating that this summary record will be cleared and rebuilt on each subsequent warehouse built. This field can be set to YES by data-load utilities specifically designed to load summary data from 3rd party data sources for the purposes of migration to SOS. If this field is set to YES it will not be cleared and rebuilt by the warehouse build process.
OUTST-VALUEdecimalOutstanding bills value (gross) for this summary level / time span
PERIOD-LEVELcharacter(2)Single character, indicating the time-span that the lock up record applies to: "Y" for whole year, "Q" for quarter, "P" for period.
Note that quarterly lock up summary data is only available for systems running 12 financial periods
PERIOD-QTRintegerA number indicating the specific quarter number or period number:
- when PERIOD-LEVEL = "Y" (year) this field is always set to 1
- when PERIOD-LEVEL = "Q" (quarter) this field contains the quarter number (1-4)
- when PERIOD-LEVEL = "P" (period) this field contains the period number (1-13)
SUMMARY-CODEcharacter(16)Used in conjunction with SUMMARY-LEVEL to indicate the specific summary level that the lock up record relates to. Depending on SUMMARY-LEVEL this field will either be:
(blank) when SUMMARY-LEVEL = "FIRM"
Branch code when SUMMARY-LEVEL = "BRANCH"
Department code when SUMMARY-LEVEL = "FEDEPT"/"MTDEPT"/"SUPERDEPT"
Fee earner code when SUMMARY-LEVEL is "FE"
Matter type code when SUMMARY-LEVEL is "MT"
Supervisor (fee earner) code when SUMMARY-LEVEL = "SUPER"
SUMMARY-LEVELcharacter(18)Set to one of eight fixed values indicating the summary level that the lock up record applies to:
"FIRM" for overall firm level
"BRANCH" for branch level
"FEDEPT" for fee earner department level
"MTDEPT" for matter type department level
"SUPERDEPT" for matter supervisor department level
"FE" for fee earner level
"“MT" for matter type level
"SUPER" for matter supervisor level
TIME-VALUEdecimalTotal value of time posted for this summary level / time span
WIP-LOCKUP-DAYSdecimalWIP lock up days for this summary level / time span
WIP-LOCKUP-ESTlogical (true/false)Set to YES if this WIP lock up days calculation on this record is an estimate
WIP-VALUEdecimalWIP value for this summary level / time span
YEARcharacter(8)Financial year of summary record
FEES-BILLEDdecimalTotal value of fees billed for this summary level / time span
BILLED-TIME-VALUEdecimalOriginal value of time that was billed for this summary level / time span
BILLED-TIME-DEFAULT-RATEdecimalThe original value of time that was billed for this summary level / time span, based on current fee earner default charge rate and excluding any charge band revaluation
BILL-REALISEdecimalThe ratio of fees billed value to the original value of time included in bills, i.e. FEES-BILLED v BILLED-TIME-VALUE. This value will be [null] when BILLED-TIME-VALUE is zero.
RACK-REALISEdecimalThe ratio of fees billed value to the value of time included in bills based on current default fee earner charge rate, i.e. FEES-BILLED v BILLED-TIME-DEFAULT-RATE. This value will be [null] when BILLED-TIME-DEFAULT-RATE is zero.

NOTE that all data in the SOS warehouse is built periodically into the warehouse from the main SOS database, and as such the warehouse contains a snapshot of live data at the point the warehouse build was last run. The warehouse does not show changes real-time as they posted into the main live SOS database.
