Data Dictionary Changes Between Vx.17 and Vx.18

This section contains details of all of the database structure changes between the x.17 and x.18 database structrure.

These changes are separated into the following sections:

Tables in x.17 which have been deleted in x.18
Tables in x.17 which have been renamed in x.18
Tables which are new in x.18 specifically for partial credit note processing
Tables which are new in x.18 for other purposes
Fields in x.17 which have been renamed in x.18
Fields in x.17 which have never been used and which are deleted in x.18


Tables Removed In Vx.18

The following V4.17 / V5.17 database tables have been removed in the V4.18 / V5.18 database structure:

  • CSB-SCHEME-STAGE
    This table was originally intended to hold links between CSB scheme and CSB stage definitions. These links have previously been held in the SUPPRELATIONS table in the x.17 schema. In this schema update the SUPPRELATIONS table has been renamed to CSB-STAGE-TO-SCHEME for clarity. This means the CSB-SCHEME-STAGE table that was originally created to hold this data is no longer required.

New Tables In Vx.18 - Specifically for credit note processing

The following database tables have been added to support new credit note functionality:

  • CREDIT-ANTITIME - Details of anticipated time that has been credit noted
  • CREDIT-COSTALLOC - Details of credited fee earner costs billed
  • CREDIT-DISB - Details of credit noted disbursements
  • CREDIT-HEADER - Credit note header record, one row per credit note (partial or full)
  • CREDIT-OLD-STYLE-ELEMENT - Holds the details of old-style bill data that has been credit noted
  • CREDIT-TIME - Details of actual time postings that have been credit noted and set back to unbilled status


New Tables In Vx.18 - Other tables

The following additional tables have been added to the x.18 database structure, some of which are reserver for future planned functionality which will be availble in future releases:

  • ACTIVATION - Holds activation and licence key information previously held in various tables including TYPEREC
  • COMPANY-STRUCTURE - Reserved for future use - intended for future reporting features
  • COMPANY-STRUCTURE-LEVEL - Reserved for future use - intended for future reporting features
  • DATA-ACCESS-GROUPS - Reserved for future use - intended for future data security features
  • ENTITY-ACCESS-GROUPS - Reserved for future use - intended for future data security features
  • LA-ACCOUNTS - Reserved for future use - intended to hold LAA account information for legal aid reporting purposes
  • DATA-SPARE11 - Reserved for future use
  • DATA-SPARE12 - Reserved for future use
  • DATA-SPARE13 - Reserved for future use
  • DATA-SPARE14 - Reserved for future use
  • MAINT-SPARE11 - Reserved for future use
  • MAINT-SPARE12 - Reserved for future use
