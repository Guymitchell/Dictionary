Practice Manager Interface




Connect Interface



Most data corresponds to the fields as labelled above in the Matter Create/Amend screens. A very few fields are updated as by-products of posting such as NO-BILLS and POST-LOCK and of course these are not directly accessible or editable. As you will see by examination of the MATTR section there is no longer any particular consistency or “specialism” between these two tables so neighbouring items on one of the Matter Enquiry tabs may well each be in different tables.

For each matter you add in Practice Manager there will be created immediately one each of MATDB, MATTR and MATAC records. The system does not wait until any posting activity actually takes place.


Field Notes

NOTES@1 contains everything you add to the Notes memo box in the Notes tab irrespective of their length.

NOTES@2 to NOTES@6 are used for the five user defined matter fields you may have created. These are the fields which if present will show in the lower portion of the Notes tab. These should not be confused with the contents of any additional bespoke fields you create in additional tabs for the matter record, which will instead be found in the MATADDDATA table.

POST_LOCK simply holds the user-id of someone in the course of making an accounts posting and so “locks” the matter ie stops someone else trying to make a posting at the same time.

If ARCHIVED-FLAG is true then DATE-CLOSED should always have a date in it as a matter can’ be archived until it has been closed, but remember that the converse is not necessarily true ie a matter may have a DATE-CLOSED value but ARCHIVED-FLAG is false.

LINK-MT-CODE should always be empty for normal ie user-created postable matters whether the matter is live, closed or archived. It will only have something in it viz the matter code itself to signal that the MATDB record is a “dummy” matter record that has been generated by Practice Manager as a by-product of creating a non-matter archive record. This feature was not introduced until patch 004 of version 2_14_0102/3_14_0102 of Practice Manager software.


Linking And Application

Simple Links

MATDB is likely to feature in most reports, almost invariably in the company of CLIDB, along with either or both of MATTR and MATAC. The following accesses the basic ingredients for any report on clients, matters and balances:

 CLIDB.CL-CODE -> MATDB.CL-CODE
 MATDB.MT-CODE -> MATAC.MT-CODE -> MATTR.MT-CODE


This variant allows you to pull the matter fee earner name in as well:

 CLIDB.CL-CODE -> MATDB.CL-CODE
 MATDB.MT-CODE -> MATAC.MT-CODE -> MATTR.MT-CODE
 MATDB.FEE-EARNER -> FEETR.FEE-EARNER


Aliased Links

Adding the FEETR table twice using alias names, you can group and subtotal the report by both the main fee earner and the matter supervisor:


 CLIDB.CL-CODE -> MATDB.CL-CODE
 MATDB.MT-CODE -> MATAC.MT-CODE -> MATTR.MT-CODE
 MATDB.FEE-EARNER -> matterFEETR.FEE-EARNER
 MATDB.SUPERVISOR -> supervisorFEETR.FEE-EARNER
