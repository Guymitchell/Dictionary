Practice Manager Interface

With the exception of the period and opening Budget Balance fields all the numeric values and the year field are updated as a by-product of accounts transaction posting and are not under normal circumstances directly accessible or editable. The other fields are copied in automatically from the parent NOMCODES record.

The opening and period Budget balances may be set initially with the following sequence:

File -> Setup -> Balance Loading -> Nominal Budgets


Number of NOMACCS records on your system

For NOMCODES where you have set the Analysis field to No there will be one NOMACCS record for each financial year of your system's operation.

For NOMCODES where you have set the Analysis field to Yes there will be one NOMACCS record for each financial year of your system’s operation for each distinct FEE-EARNER/MT-TYPE combination. Typically the commonest NOMCODES set with Analysis on are the controls X05, X08 and X20 representing Unbilled Disbursements, Costs Billed and Unpaid Bills. For even a small to medium sized firm a quick browse through NOMACCS will probably reveal hundreds of same-year NOMACCS records for each of these NOMCODES.


Field Notes

FEE-EARNER, MT-TYPE, MAT-GRP, YEAR, CODE and BRANCH will be written into NOMTRANS records and hence into NOMACCS automatically. You cannot access these fields in NOMACCS directly.

Note that the CLOSING-BAL and CLOSING-BUDG fields always contain cumulative actual or budget balances to date but not to any given historic period end i.e. if it is now Period 7 there is no immediate figure available for a balance as at the end of say Period 4. To report on historic information you will therefore have to create formulae to calculate these intermediate cumulative balances.

For example to derive the end of Period 4 actual balance you might create a formula called YTDPeriod4 consisting of the following expression:

NOMACCS.OPENING-BAL + NOMACCS.PERIOD-BAL@1 + NOMACCS.PERIOD-BAL@2 + NOMACCS.PERIOD-BAL@3 + NOMACCS.PERIOD-BAL@4


Linking And Application

NOMACCS will be useful in a wide variety of management reports as it summarises both nominal and client matter financial activity. It would usually be linked to NOMCODES on the CODE field to provide an actual nominal analysis description. It is also often linked to NOMTRANS if you want to find and group client matter accounts postings by their cashbook ie CODE instead of by their client matter code.

The compound indexes shown above offer an easy way to produce highly focussed data on particular nominal codes.

For example to produce a quick report of costs delivered by your second branch for 2001 you could set a record selection formula which included:

 NOMACCS.CODE =”X08” and
 NOMACCS.YEAR =” 2001” and
 NOMACCS.BRANCH =” 002”

Remember that where a nominal code has Analysis set to Yes then your report will have to group the nominal accounts data by their CODE and that the required total for the code must be placed in that group’s heard or footer. The underlying data for each fee/earner/matter type “sub account” would be placed in the report’s Detail section and quite possibly suppressed or at least hidden from view. This is exactly what happens when you run the standard Practice Manager Trial Balance at Code level – the single line nominal code totals that you see are actually sitting in Group headers, and if you drill down you will find, certainly in case of codes like X08 or X20, line after line of detail per fee earner and matter type for the same nominal code.
