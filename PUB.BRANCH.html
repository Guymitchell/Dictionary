Practice Manager Maintenance Screen



BRANCH corresponds to the data entered in the BRANCH maintenance program but, behind the scenes, it is also read in by the software every time a client matter is added to the system and every time a posting is made.  Once entered at the very outset of using Practice Manager the information held is unlikely to be amended except at times of other major changes in the firm and it is therefore very much a “write once, read often” table.



Field Notes

CODE holds the branch code itself, though in all the many other tables that hold the branch code the field is called BRANCH.

COUNTRY governs the display or suppression of extra accounting data for Scottish branches.  The Accounts Matter reports and the Accounts Batch listings from the standard Practice Manager report menus all contain an extra specialized Client Control summary if any one or more branches of a firm have Scotland in this field. For these purposes Scotland is uniquely different, so England should be the setting for any jurisdiction other than Scotland.

LA-ACC holds the original code that applied for all purposes. It corresponds to Legal Aid Ac/No on the Branch maintenance screen.

CONTRACTNO derives from LA-ACC and should now be used by the branch for its civil contract code. It corresponds to Contract No on the Branch maintenance screen.

LA-REF also derives from LA-ACC and should now be used by the branch for its CDS (criminal) code. It corresponds to Supplier No on the Branch maintenance screen.

CIVIL-SCHEDULE-NO holds the Civil Legal Aid schedule number for the branch.

CRIMINAL-SCHEDULE-NO holds the Criminal Legal Aid schedule number for the branch.

Note that the branch telephone, fax, email and mobile numbers are not stored in BRANCH table but are stored in the PUB.TELEPHONES table with PUB.TELEPHONES.TYPE = "BR" and PUB.TELEPHONES.REFERENCE holding the branch code.



Suggested Linking

For multi-branch firms BRANCH is the only table which contains the branch name for display on reports, so common links might be:

To list matters by branch:

PUB.MATDB.BRANCH -> PUB.BRANCH.CODE


To list individual accounts postings by originating branch:

PUB.TRTRANS.BRANCH -> PUB.BRANCH.CODE


Where your report contains more than one table with a BRANCH field take care in deciding which table to link to BRANCH as the results may differ.

BRANCH has also been used in some standard reports where reference is needed to period end dates in previous accounting years before the current one.
