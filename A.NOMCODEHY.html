Practice Manager Interface

This table holds nominal code folder layer and location information as defined in Practice Manager Chart of Accounts.



Field Notes

BAL-PL cannot be # alone as this symbol demands a suffix to indicate the type of bank, i.e. "C", "D" or "O" to signal "Client", "Deposit" or "Office".

DR-CR simply defines whether a nominal is usually a debit or credit item.

Print Line To display as Set up in Chart of Accounts; sort data based on Print Line fields.

Linking and Application

NOMCODEHY usually partners NOMACCS and other tables such as NOMTRANS and is so used to show the folder system for nominal codes.


This example could provide a list of postings grouped and totalled by analysis code and/or cashbook:

NOMTRANS.NL-CODE -> NOMCODEHY.CONNECTING-NOMCODE


This example ignores details and provides a summary of balances by analysis code and/or cashbook:

NOMACCS.NL-CODE -> NOMCODEHY.CONNECTING-NOMCODE
