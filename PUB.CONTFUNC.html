User Interface Notes

This table is updated as a by-product of assigning a function to an individual contact when you are working in the Relationships tab of an individual contact record. The options to add or remove a functional class for a contact as shown below are accesses by right-clicking on the functions list:



Note that there can be multiple records containing the same CON.REFERENCE viz one per function per contact. It is not compulsory to assign a function to a contact so records may not be present for every individual contact and care must be taken if linking to CONTACTS.



Linking And Application

A list of contacts sorted and grouped by their function, or a list of contacts not assigned a function:

CONTACTS.CON-REFERENCE -> CONTFUNC.CON-REFERENCE (with left outer join)
CONTFUNC.FUN-CODE -> FUNCTIONS.FUN-CODE (to provide the function description)
