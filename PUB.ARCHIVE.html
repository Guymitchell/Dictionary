Field Notes

For archived client matters most of these fields are populated automatically when the matter is archived based on values stored in MATDB and MATTR records.

TYPE is always “FILE” for records created automatically as a by-product of the matter archiving procedure.

The NOTES, LOCATION and any or all UDF fields can be added later to, or edited on, all archive records whether they have been created automatically from matter records or added “manually” as non-matter archive types such as deeds or wills.


User Defined Fields

These must be created and labelled before they can be used. Subject to access rights you may add any or all of the three groups of five available fields with the following Practice Manager menu structure viz:

File -> Setup -> Preferences -> Archives

The maximum number of fields is five of each type viz dates text and numbers whether integer or decimal. Unused fields of one type can’t be carried over to another type, e.g. you can’t have six text fields, six number fields and three date fields.


Suggested Linking

Linking client records to their archive records:

A.CLIDB.CL-CODE -> A.ARCHIVE.CL-CODE

Reporting on archived matters for selected fee earners:

PUB.FEETR.FEE-EARNER -> A.ARCHIVE.FEE-EARNER

Reporting on historic time postings for an archived matter:

A.ARCHIVE.MT-CODE -> PUB.TRTRANS.MT-CODE

Linking the various ARCHIVE-UDF label fields to provide column headers for a report showing user-defined field data:

A.ARCHIVE.TYPE -> PUB.ARCHIVE-UDF.TYPE
