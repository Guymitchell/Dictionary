User Interface Notes

SOLLEDGER contains transaction details generated by Anticipated Disbursement postings. So long as these disbursements are merely "Pencilled" they will be in SOLLEDGER only, not in NOMTRANS, so fields marked above as being the same as a particular NOMTRANS field will be blank or zero. Once posted however, a NOMTRANS record will be created from the above and in turn certain NOMTRANS data such as the transaction and entry numbers will be copied into the existing SOLLEDGER record.


Linking And Application

Once an anticipated disbursement has been posted, all relevant details will be in the NOMTRANS and PURCHASES tables, so the use of SOLLEDGER is probably best confined to listings or analysis of "pencilled" anticipated disbursements only. MT-CODE and CL-CODE provide the usual links to the MATDB and CLIDB tables respectively.
